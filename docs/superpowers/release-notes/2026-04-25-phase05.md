# Phase 0.5 — Foundations Release Notes

## What's new

- **`armor` and `weapon` modules.** The ` ```armor ` and ` ```weapon ` code fences in compendium markdown now render proper stat blocks. The PC entity registry can resolve armor and weapon slugs to typed entities, unblocking SP5's AC and attack derivation.
- **Magic-item schema upgrade.** Optional structured fields for mechanical bonuses (`bonuses.ac`, `bonuses.weapon_attack`, etc.), charges with recharge formulas, attached spells (`attached_spells.charges`/`daily`/`will`), structured attunement requirements, immunities/resistances, container capacity, light emission, and more. **Existing prose-only magic items render unchanged.**
- **Formula-tag system rewrite.**
  - `atk:K` now means "ability mod alone" (no proficiency). Previously, this implicitly added PB.
  - `atk:K+PB` means "ability mod + proficiency". Use this for proficient attacks.
  - `attack:` and `damage:` are accepted as input aliases for `atk:` and `dmg:`. Canonical emit is `atk:` and `dmg:`.
  - `dc:K` always implicitly includes proficiency. Writing `dc:K+PB` is a syntax error.
- **SRD monster compendium regenerated** with PB-aware tags. Run `npm run regenerate-srd-monsters` to refresh.
- **PC recalc** now aggregates weapon/armor/tool/language proficiencies across class + race + background + feats. New query helpers `isProficientWithWeapon` and `isProficientWithArmor` (used internally; SP5 will surface them in attack-row rendering).

## Migration notes (homebrew users)

If you have homebrew monsters in `Me/Monsters/` that use `atk:K` tags:

```bash
npm run migrate-formula-tags ~/Documents/V/Compendium/Me/Monsters
```

This rewrites all `atk:`/`dc:`/`damage:`/`dmg:` tags in your homebrew using the same damage-driven attribution as the SRD regeneration. **Without this migration**, your homebrew monster attack values will display incorrectly (specifically: too low by your monster's PB for attacks that were proficient under the old system).

If you've authored magic items with structured fields (e.g., for SP4-era homebrew), you don't need to do anything — Phase 0.5 only adds optional fields; existing files are forward-compatible.

## What's next

SP5 (Equipment & Inventory Engine) is unblocked. It will use the new armor/weapon modules, the proficiency aggregation, and the proficiency-aware tag emission to derive AC, attack rows, and inventory mechanics on character sheets.
